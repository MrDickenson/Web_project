Бізнес-документ проекту "Simple ToDo System"

1. Назва системи й короткий опис

Назва: Simple ToDo System

Simple ToDo System — це легкий веб-додаток, призначений для індивідуального управління завданнями. Система дозволяє користувачам швидко фіксувати свої плани, відстежувати їх виконання та організовувати свій робочий день. Головний акцент зроблено на швидкодії та простоті інтерфейсу, щоб мінімізувати час на внесення даних.

Проект розробляється як клієнт-серверна архітектура, де веб-інтерфейс (Frontend) взаємодіє з сервером (Backend) через REST API. Це забезпечує гнучкість системи та можливість у майбутньому легко замінити клієнтську частину або додати мобільний додаток без змін у серверній логіці.

2. Користувачі (ролі)

Зареєстрований користувач (User): Основна роль. Може створювати особисті завдання, редагувати їх статус, видаляти виконані пункти. Бачить тільки власні дані.

3. Основні сценарії використання (Use Cases)

Перегляд списку справ: Користувач заходить на сторінку і бачить актуальний список своїх завдань з їх статусами (виконано/не виконано).

Створення завдання: Користувач вводить текст завдання, натискає "Додати", і воно з'являється у списку та зберігається в базі даних.

Зміна статусу: Користувач відмічає завдання як "виконане", після чого воно візуально змінюється (наприклад, закреслюється).

Видалення завдання: Користувач видаляє помилкове або застаріле завдання зі списку.

Фільтрація: (Майбутній сценарій) Користувач переглядає тільки активні або тільки виконані завдання.

4. Функціональні вимоги

Система повинна надавати REST API для роботи із завданнями (GET, POST, PUT, DELETE).

Система повинна зберігати дані у реляційній базі даних (SQLite).

Система повинна валідувати вхідні дані (наприклад, не можна створити пусте завдання).

5. Нефункціональні вимоги

Швидкодія: Час відповіді сервера на запити API не повинен перевищувати 200 мс при нормальному навантаженні.

Зручність: Інтерфейс має бути інтуїтивно зрозумілим без додаткового навчання.

6. Обмеження

Система не буде підтримувати роботу в офлайн-режимі на першому етапі.

Система не буде мати складних функцій командної роботи (призначення завдань іншим людям) у версії MVP (Minimum Viable Product).

Система не передбачає інтеграції зі сторонніми календарями (Google Calendar) у поточному релізі.
